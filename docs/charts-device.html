HNSW обгоняет Linear Scan начиная с 5K векторов — P95 9.62 мс vs 10.44 мс. При 10K — всего 14 мс на запрос. Типичная фотогалерея (5-15K фото) получает поиск быстрее, чем один кадр анимации (16 мс).

HNSW Scaling Curve on Device

Как растёт латентность HNSW с ростом коллекции — P50 и P95
Даже P95 при 10K векторов (14.16 мс) укладывается в бюджет одного кадра (16 мс). Сублинейный рост подтверждает O(log n) сложность HNSW.

Batch Throughput (10K vectors)

HNSW — компромисс: быстрый поиск за счёт медленной индексации. При 53 vec/s индексация 10K фото занимает ~3 мин — приемлемо для фоновой задачи через WorkManager. LinearScan индексирует мгновенно, но поиск O(n) не масштабируется.

Latency Distribution at Each Scale

Сравнение P50, Avg и P95 для HNSW — разброс между median и tail
Разброс P50-P95 растёт с размером индекса: при 1K — всего 1.35 мс, при 10K — 5.76 мс. Это ожидаемо: чем больше граф, тем выше вариативность длины пути обхода.
